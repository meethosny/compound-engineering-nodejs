Commands Are Now Skills

The plugin no longer ships a separate set of slash commands. Every former /command was migrated into a skill -- it lives under skills/<name>/SKILL.md and works identically as a slash command in Claude Code. There's no separate "commands" count anymore: the workflow commands are skills.

Nothing changes for you at the keyboard

You still type /js-ce:plan, /js-ce:review, and /js-ce:work exactly as before. The difference is internal: these are defined as skills, so they're documented on the Skills page rather than a standalone commands page.

Core Workflow Commands

The eight core workflow skills keep the colon namespace (js-ce:) so they read as slash commands and don't collide with Claude Code's built-in /plan and /review.

Slash command What it does
/js-ce:ideateGenerate and critically evaluate grounded ideas before brainstorming one in depth
/js-ce:brainstormExplore requirements and approaches through dialogue, then write a requirements doc
/js-ce:planCreate structured plans for any multi-step task; also deepen existing plans
/js-ce:reviewStructured code review with tiered persona agents, confidence gating, and dedup
/js-ce:workExecute work efficiently while maintaining quality and finishing features
/js-ce:work-betaSame as work, with experimental Codex delegation mode
/js-ce:compoundDocument a recently solved problem to compound team knowledge
/js-ce:compound-refreshRefresh stale learning docs under docs/solutions/

Everything Else

All other former commands -- debugging, git, todos, testing, docs, release, and more -- are now hyphenated skills invoked as /js-ce-<name> or skill: js-ce-<name>. See the full list, grouped by category, on the Skills page.

Skill Reference

Think of skills as reference manuals that Claude Code can read mid-conversation. When you're writing Node.js code and want a modern style, or building an npm package like Sindre Sorhus would, you don't need to paste documentation—just invoke the skill. Claude reads it, absorbs the patterns, and writes code that way.

How to Use Skills

# In your prompt, reference the skill
skill: js-[skill-name]

# Examples
skill: js-modern-nodejs-style
skill: js-gemini-imagegen
skill: js-create-agent-skills

Skills vs Agents

Agents are personas—they do things. Skills are knowledge—they teach Claude how to do things. Use claude agent js-[name] when you want someone to review your code. Use skill: js-[name] when you want to write code in a particular style yourself.

Architecture & Design (2)

High-level thinking skills. Use these before writing code—when you need to decide what to build, how to structure it, or whether to use a prompt-native approach instead of traditional code.

js-agent-native-architecture

Architecture

Build AI agents using prompt-native architecture where features are defined in prompts, not code. Instead of writing workflow functions for an agent to call, you define outcomes in natural language and give the agent primitive tools to figure out the how. This skill covers the full philosophy: features as prompts, tools as primitives, and optional self-modification.

Capabilities

  • Prompt-native architecture design and planning
  • MCP primitive tool design (read, write, store, call)
  • System prompt authoring for agent behavior
  • Self-modification with approval gates and rollback
  • Refactoring existing code to prompt-native patterns

Key Principle

Whatever the user can do, the agent can do. Features are prompts that define outcomes. Tools provide capability, not behavior.

skill: js-agent-native-architecture

js-brainstorming

Planning

Guide for exploring user intent, approaches, and design decisions before planning. Use this before implementing features or making changes when requirements are unclear. It walks through four phases: assess clarity, understand the idea, explore approaches, and capture the design—all with a YAGNI mindset.

Four Phases

  1. Assess Clarity - Determine if brainstorming is even needed
  2. Understand the Idea - Ask questions one at a time to clarify intent
  3. Explore Approaches - Propose 2-3 concrete approaches with trade-offs
  4. Capture the Design - Summarize key decisions in a structured doc

Integration

  • Outputs to docs/brainstorms/ directory
  • Hands off to /js-workflows:plan for implementation details
skill: js-brainstorming

Development Tools (7)

These skills teach Claude specific coding styles and architectural patterns. Use them when you want code that follows a particular philosophy—not just any working code, but code that looks like it was written by a specific person or framework.

js-sindre-sorhus-package-writer

npm

Write npm packages following Sindre Sorhus's proven patterns from 1000+ packages with billions of downloads. Every package follows the same structure: ESM-first, zero runtime dependencies, TypeScript definitions always included, and clear README with examples. When you're building an npm package and want it to feel production-ready from day one, this is how you do it.

Philosophy

  • Do one thing well (single-purpose packages)
  • Zero or minimal dependencies
  • ES Modules first, always include .d.ts
  • Smart defaults, simple configuration

Key Patterns

  • Named exports over default exports
  • Input validation with fast-fail TypeError
  • Node.js built-in test runner (no frameworks)
  • CLI pattern with minimal argument parsing
  • Semantic versioning, strict semver

Reference Files

  • references/module-organization.md
  • references/testing-patterns.md
  • references/resources.md
skill: js-sindre-sorhus-package-writer

js-compound-docs

Docs

You just fixed a weird build error after an hour of debugging. Tomorrow you'll forget how you fixed it. This skill automatically detects when you solve something (phrases like "that worked" or "it's fixed") and documents it with YAML frontmatter so you can find it again. Each documented solution compounds your team's knowledge.

Auto-Triggers

Phrases: "that worked", "it's fixed", "working now", "problem solved"

7-Step Process

  1. Detect confirmation phrase
  2. Gather context (module, symptom, investigation, root cause)
  3. Check existing docs for similar issues
  4. Generate filename
  5. Validate YAML frontmatter
  6. Create documentation in category directory
  7. Cross-reference related issues

Categories

  • build-errors/
  • test-failures/
  • runtime-errors/
  • performance-issues/
  • database-issues/
  • security-issues/
skill: js-compound-docs

js-frontend-design

Design

Create distinctive, production-grade frontend interfaces with high design quality. You've seen what AI usually generates: Inter font, purple gradients, rounded corners on everything. This skill teaches Claude to design interfaces that don't look like every other AI-generated site. It's about purposeful typography, unexpected color palettes, and interfaces with personality.

Design Thinking

  • Purpose - What is the interface for?
  • Tone - What feeling should it evoke?
  • Constraints - Technical and brand limitations
  • Differentiation - How to stand out

Focus Areas

  • Typography with distinctive font choices
  • Color & theme coherence with CSS variables
  • Motion and animation patterns
  • Spatial composition with asymmetry
  • Backgrounds (gradients, textures, patterns)

Avoids generic AI aesthetics like Inter fonts, purple gradients, and rounded corners everywhere.

skill: js-frontend-design

js-modern-nodejs-style

Node.js

Write Node.js code following modern, pragmatic patterns inspired by TJ Holowaychuk (minimalism), Matteo Collina (performance), and Sindre Sorhus (package authoring). Express routes that are simple and predictable. Middleware that does one thing well. Async-first, MVP-focused, minimal dependencies.

Key Patterns

  • Single-responsibility middleware - One middleware, one task
  • Short route handlers - 5-15 lines, extract business logic to modules
  • Factory functions - Named exports, explicit dependencies
  • Centralized error handling - Custom error classes, single handler
  • Async/await everywhere - No callbacks, Promise.all for parallel ops

Philosophy

  • Minimalism: small modules, minimal dependencies
  • Composability: build from interchangeable parts
  • Async-first: never block the event loop
  • Ship fast: MVP first, enhance based on feedback
  • Explicit over magic: clear code over clever code
skill: js-modern-nodejs-style

Content & Workflow (6)

Writing, editing, reviewing, and organizing work. These skills handle everything from style guide compliance to git worktree management and PR comment resolution—the meta-work that makes the real work easier.

js-document-review

Review

Structured self-review for brainstorm and plan documents. Assesses five dimensions—clarity, completeness, specificity, YAGNI compliance, and consistency—then auto-fixes minor issues (typos, formatting) and asks approval for substantive changes. Use as a quality gate before starting implementation.

Review Dimensions

  • Clarity - Understandable without additional context
  • Completeness - All necessary sections and edge cases
  • Specificity - Measurable criteria, concrete details
  • YAGNI - No unnecessary features or abstractions
  • Consistency - Formatting, frontmatter, references

Grading System

  • PASS - Meets standard, no changes needed
  • MINOR - Small issues, auto-fixable
  • NEEDS WORK - Substantive gaps, requires user input
skill: js-document-review

js-every-style-editor

Content

Review copy for Every's style guide compliance. Does a four-phase line-by-line review: context assessment, detailed line edits, mechanical checks, and actionable recommendations. It's like having a copy editor who never gets tired—checking grammar, punctuation, capitalization, and formatting consistency.

Four-Phase Review

  1. Initial Assessment - Context, type, audience, tone
  2. Detailed Line Edit - Sentence structure, punctuation, capitalization
  3. Mechanical Review - Spacing, formatting, consistency
  4. Recommendations - Actionable improvement suggestions

Style Checks

  • Grammar and punctuation
  • Style guide compliance
  • Capitalization rules
  • Word choice optimization
  • Formatting consistency
skill: js-every-style-editor

js-file-todos

Workflow

File-based todo tracking system in a todos/ directory with status, priority, and dependencies. Each filename encodes the metadata. No database, no UI, just markdown files with YAML frontmatter. When you need to track work without setting up Jira, this is the system.

File Format

# Naming convention
{issue_id}-{status}-{priority}-{description}.md

# Examples
001-pending-p1-security-vulnerability.md
002-ready-p2-performance-optimization.md
003-complete-p3-code-cleanup.md

Status Values

  • pending - Needs triage
  • ready - Approved for work
  • complete - Done

Priority Values

  • p1 - Critical
  • p2 - Important
  • p3 - Nice-to-have
skill: js-file-todos

js-git-worktree

Git

Manage Git worktrees for isolated parallel development with a KISS interface. You're working on a feature branch, but you need to review a PR without losing your current work. Git worktrees let you have multiple branches checked out simultaneously in separate directories. This skill manages them—create, switch, cleanup—so you can context-switch without stashing.

Commands

# Create new worktree
bash scripts/worktree-manager.sh create feature-login

# List worktrees
bash scripts/worktree-manager.sh list

# Switch to worktree
bash scripts/worktree-manager.sh switch feature-login

# Clean up completed worktrees
bash scripts/worktree-manager.sh cleanup

Integration

  • Works with /js-workflows:review for isolated PR analysis
  • Works with /js-workflows:work for parallel feature development

Requirements

  • Git 2.8+ (for worktree support)
  • Worktrees stored in .worktrees/ directory
skill: js-git-worktree

js-resolve-pr-parallel

PR

Resolve all PR review comments in parallel using multiple js-pr-comment-resolver agents. After receiving code review feedback, this skill analyzes unresolved comments, groups them by type (code changes, questions, suggestions), then spawns a parallel agent for each comment thread for maximum speed.

Workflow

  1. Analyze - Get all unresolved comments via gh pr view
  2. Plan - Group by type: code changes, questions, suggestions
  3. Implement - Spawn js-pr-comment-resolver agents in parallel
  4. Commit & Resolve - Stage, commit, push, and verify all addressed

When to Use

  • After receiving code review feedback on a PR
  • When there are multiple unresolved comment threads
  • To efficiently address all review comments at once
skill: js-resolve-pr-parallel

Multi-Agent Orchestration (1)

Coordinate multiple AI agents working together. When a single agent isn't enough, orchestrate a swarm of specialists that communicate, share tasks, and self-organize.

js-orchestrating-swarms

Swarms

Orchestrate multi-agent swarms using Claude Code's TeammateTool and Task system. This comprehensive skill covers everything from spawning parallel specialist reviewers to building sequential pipelines with dependencies to self-organizing task pools. A leader creates a team, spawns teammates, coordinates via inbox messages, and manages graceful shutdown.

Orchestration Patterns

  • Parallel Specialists - Multiple reviewers analyze code simultaneously
  • Pipeline - Sequential stages with auto-unblocking dependencies
  • Swarm - Self-organizing workers that race to claim tasks
  • Research + Implementation - Synchronous research, then guided implementation
  • Plan Approval - Require leader approval before execution
  • Coordinated Refactoring - Multi-file changes with dependency tracking

Core Primitives

  • Team - Named group of agents with leader and teammates
  • Task - Work items with status, owner, and dependencies
  • Inbox - JSON-based inter-agent messaging
  • Backends - in-process, tmux, or iTerm2 spawn modes

TeammateTool Operations

  • spawnTeam, discoverTeams, requestJoin
  • write (message one), broadcast (message all)
  • requestShutdown, approveShutdown, cleanup
  • approvePlan, rejectPlan
skill: js-orchestrating-swarms

Browser Automation (1)

Automate web browsers for testing, scraping, and form interaction—all through CLI commands.

js-agent-browser

Browser

Browser automation using Vercel's agent-browser CLI for web interaction, forms, screenshots, and scraping. Uses ref-based element selection (@e1, @e2) from accessibility snapshots. Navigate to a page, take a snapshot to get interactive elements with refs, then interact using those refs—all through Bash commands.

Core Workflow

  1. Navigate - agent-browser open <url>
  2. Snapshot - agent-browser snapshot -i to get refs
  3. Interact - agent-browser click @e1 / fill @e2 "text"
  4. Re-snapshot - After navigation or DOM changes

Capabilities

Navigate, click, fill, select, scroll
Screenshots and PDF export
Semantic locators (role, text, label)
Parallel sessions for multiple browsers
JSON output for structured parsing

Requirements

  • npm install -g agent-browser
  • agent-browser install (downloads Chromium)
skill: js-agent-browser

Cloud & File Transfer (1)

Move files to and from cloud storage providers without writing custom upload code.

js-rclone

Cloud

Upload, sync, and manage files across cloud storage providers using rclone. Supports S3, Cloudflare R2, Backblaze B2, Google Drive, Dropbox, DigitalOcean Spaces, and any S3-compatible storage. Handles setup checks, provider configuration, single file uploads, directory syncing, and large file chunked transfers.

Supported Providers

Provider Type Auth
AWS S3 s3 Access key + secret
Cloudflare R2 s3 Access key + endpoint
Backblaze B2 b2 Account + app key
Google Drive drive OAuth flow
Dropbox dropbox OAuth flow

Key Operations

  • rclone copy - Upload files/directories
  • rclone sync - Mirror with deletions
  • rclone ls / lsd - List contents
  • rclone check - Verify uploads
  • Large file chunked uploads with --s3-chunk-size
skill: js-rclone

Image Generation (1)

Generate images with AI. Not stock photos you found on Unsplash—images you describe and the model creates.

js-gemini-imagegen

AI Images

Generate and edit images using Google's Gemini API. Need a logo with specific text? A product mockup on a marble surface? An illustration in a kawaii style? You describe what you want, it generates it. Supports text-to-image, image editing, multi-turn refinement, style transfers, and composition from multiple reference images.

Features

Text-to-image generation
Image editing & manipulation
Multi-turn iterative refinement
Multiple reference images (up to 14)
Google Search grounding (Pro)

Default Model

Model Resolution Best For
gemini-3-pro-image-preview 1K-4K All image generation (default)

Available Settings

  • Resolutions: 1K (default), 2K, 4K
  • Aspect Ratios: 1:1, 2:3, 3:2, 3:4, 4:3, 4:5, 5:4, 9:16, 16:9, 21:9

Quick Start

# Text-to-image
python scripts/generate_image.py "A cat wearing a wizard hat" output.png

# Edit existing image
python scripts/edit_image.py input.png "Add a rainbow in the background" output.png

# Multi-turn chat
python scripts/multi_turn_chat.py

Prompting Best Practices

  • Photorealistic - Include camera details: lens type, lighting, angle, mood
  • Stylized Art - Specify style explicitly: kawaii, cel-shading, bold outlines
  • Text in Images - Be explicit about font style and placement (use Pro model)
  • Product Mockups - Describe lighting setup and surface

Requirements

GEMINI_API_KEY Required environment variable
google-genai Python package
pillow Python package for image handling

All generated images include SynthID watermarks. Image-only mode won't work with Google Search grounding.

skill: js-gemini-imagegen

Agentless Architecture

The plugin ships 0 standalone agents. There is no agents/ directory and nothing to invoke with claude agent <name>. The reviewer, researcher, and reviewer-lens specialists you may remember still exist — but they now live inside the skills that use them, as prompt-only personas dispatched to generic subagents.

Looking for a specific reviewer?

Specialists aren't listed here because they aren't independently installable components. Run the skill that owns them — for example /js-ce:code-review or js-ce-doc-review — and it dispatches the right personas for you. See the Skills reference for all 29 skills.

What "Agentless" Means

Every capability the plugin offers is packaged as a skill. A skill is self-contained: its SKILL.md holds the workflow, and its references/ directory holds everything the workflow needs, including specialist prompts.

  • No top-level agents. Nothing is registered as a separate agent, so there is no agent catalog and no agent count to keep in sync.
  • Personas are prompt assets. A reviewer or researcher is just a markdown file with a prompt — no YAML frontmatter, an unprefixed filename like correctness-reviewer.md — stored under the calling skill's references/personas/ or references/agents/.
  • Skills stay portable. Because each skill carries its own specialists, it converts to other agent tools as a single unit with no separate agent-registration step.

How Specialists Run

When a skill needs an expert opinion, it reads the persona prompt from its own references/ directory and dispatches a generic subagent with that prompt. The subagent isn't a named agent — it's a plain worker running the persona's instructions. Several can run in parallel, and the calling skill merges, de-duplicates, and confidence-gates their findings. 

skills/js-ce-code-review/
├── SKILL.md                         # the review workflow
└── references/
    └── personas/
        ├── correctness-reviewer.md   # dispatched as a generic subagent
        ├── security-reviewer.md
        ├── performance-reviewer.md
        └── ...                        # selected by what the diff touches

The persona prompt travels with the skill. If two skills need the same kind of specialist, each keeps its own copy — skills never reach outside their own directory.

Examples

A few skills that dispatch specialist personas rather than calling standalone agents:

Skill Dispatches personas like
/js-ce:code-review correctness, maintainability, testing, and project-standards reviewers (always on), plus conditional reviewers — security, performance, migrations, CLI, async-UI, Swift — chosen by what the diff touches
js-ce-doc-review coherence, feasibility, design, product, scope, security, and adversarial document review lenses
/js-ce:plan research personas that read framework docs, git history, prior learnings, and the open web
js-ce-resolve-pr-feedback a PR-comment-resolver persona that evaluates and fixes review threads in parallel

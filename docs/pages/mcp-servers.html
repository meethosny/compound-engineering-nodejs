MCP Servers Reference

The js-compound-engineering plugin ships zero MCP servers. It used to bundle Context7 for documentation lookups, but that dependency was removed so the plugin stays self-contained and portable across agent platforms. Nothing to configure, nothing that can fail to auto-load.

Context7 is no longer bundled

Earlier versions shipped the Context7 MCP server. It has been removed. If you relied on it, you can still add it yourself in a few seconds — see Add Your Own below.

No Bundled Servers

There are no MCP servers to document. The plugin's value lives entirely in its 29 self-contained skills, each of which carries the prompts and scripts it needs. Removing the bundled Context7 server means:

  • No MCP endpoint to configure, authenticate, or troubleshoot on install
  • No network dependency required for the plugin to work
  • Cleaner conversion when porting skills to other agent tools (opencode, codex, cursor, and the rest)

If a skill needs live documentation, it uses whatever documentation tools your environment already provides. You are also free to add any MCP server you like — including Context7 — to your own Claude Code config.

Add Your Own

Want documentation lookups or any other capability back? Adding an MCP server to Claude Code is a two-minute copy-paste job — and it lives in your config, not the plugin. The examples below use a placeholder HTTP server; swap in the name and URL of whichever server you want (a docs service, a database bridge, an internal tool).

Project-Level Configuration

To enable for just this project, add this to .claude/settings.json in your project root:

{
  "mcpServers": {
    "your-server": {
      "type": "http",
      "url": "https://mcp.example.com/mcp"
    }
  }
}

Global Configuration

Or enable everywhere—every project on your machine gets this server. Add to ~/.claude/settings.json:

{
  "mcpServers": {
    "your-server": {
      "type": "http",
      "url": "https://mcp.example.com/mcp"
    }
  }
}

Verifying MCP Servers

After you add the config, restart Claude Code. Then confirm the server is wired up:

# Ask Claude what it has
"What MCP tools do you have access to?"

# If it fails, double-check your JSON syntax and file paths

Why Your Third Code Review Should Be Easier Than Your First

Think about the last time you fixed an N+1 query or a race condition in your Node.js app. You found it. You fixed it. Then next month, different developer, same bug, same investigation. That's linear engineering—you solved it, but the solution evaporated.

"Most engineering work is amnesia. You solve a problem on Tuesday, forget the solution by Friday, and re-solve it next quarter. Compounding engineering is different: each solved problem teaches the system. The security review you run today makes tomorrow's review smarter. The pattern you codify this sprint prevents bugs in the next three."

Frequently Asked Questions

What is Compounding Engineering?

It's the opposite of how most teams work. Normally, you fix a bug, ship it, and forget it. Next month someone hits the same bug and re-solves it from scratch. Compounding engineering means each fix teaches the system. Your third code review is faster than your first because the agents learned patterns. Your tenth security audit catches issues you missed in audit #2. The work accumulates instead of evaporating.

How do agents differ from skills?

Agents are coworkers with specific jobs. The js-ce-security-reviewer does security reviews. The js-ce-modern-nodejs-reviewer enforces Node.js conventions. They're usually dispatched by workflow skills, but you can call any directly: claude agent js-ce-security-reviewer.

Skills package the plugin's workflows and expertise. The js-ce-modern-nodejs-style knows pragmatic Node.js patterns; the js-ce-gemini-imagegen knows how to generate images. Claude invokes them automatically when relevant, or you can call them explicitly: skill: js-ce-modern-nodejs-style. The workflow commands are skills too—run them as slash commands like /js-ce:plan.

Why isn't the MCP server loading automatically?

Usually it just works. If Context7 can't fetch docs, manually add the server to your .claude/settings.json file—check the README or the MCP servers page for copy-paste config. Takes 30 seconds and you're done.

What languages does this support?

This fork is optimized for Node.js, TypeScript, and Python. The workflow skills, research agents, and skills like js-ce-gemini-imagegen work with any language. The js-ce-security-reviewer finds SQL injection whether it's in Express, Django, or Laravel.

How do I create my own agents or skills?

Drop a new agent in agents/<category>/js-ce-<name>.md or a skill in skills/js-ce-<name>/SKILL.md with the required frontmatter, then add it to the README and bump the counts. Use a skill-authoring helper to scaffold the SKILL.md, and follow the plugin's skill compliance checklist. When you remove a component, update the cleanup registry so the cleanup CLI can back up stale artifacts.
